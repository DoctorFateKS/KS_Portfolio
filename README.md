### About

A **production-grade personal portfolio** built with Ruby on Rails, designed to showcase my expertise in Ruby, Rails, software architecture, and product-oriented development.

This project is not just a static portfolio, it is intentionally treated as a **real-world Rails application**, following best practices in code qualit, performance, and maintainability.

---

### Purpose

The goals of this project are to:

- Demonstrate **deep Ruby & Rails expertise**
- Showcase **clean architecture and high-quality code**
- Highlight real-world **product and engineering thinking**
- Serve as a reference implementation for a modern Rails portfolio
- Act as a living, evolving project rather than a one-off website

---

### Philosophy

> *A portfolio should reflect how you build production software, not just how it looks.*

This project emphasizes:

- Readability over cleverness  
- Convention over configuration  
- Explicit architecture and boundaries   
- Long-term maintainability  

---

## Tech Stack

### Backend

- **Ruby** (latest stable)
- **Ruby on Rails** (latest stable)
- PostgreSQL
- Sidekiq + Redis (background jobs)

### Frontend

- Hotwire (Turbo + Stimulus)
- HTML5 / CSS3
- Modern JavaScript (ES6+)

### DevOps & Tooling

- RuboCop
- Brakeman

---

### Application Structure

The application follows standard Rails conventions, with additional emphasis on:

- Clear domain boundaries
- Service objects where appropriate
- Minimal controllers
- Fat models only when it makes sense
- Explicit naming and intent


---

### Main Features

- Landing page with clear positioning
- About section with professional background
- Detailed **project case studies**
- Technical blog with long-form articles
- Open source & GitHub integration
- Contact form with validation
- SEO-friendly and accessible markup
- Dark mode support

---

### Security & Best Practices

- CSRF protection enabled
- Secure headers
- Input validation
- Background jobs isolated
- No sensitive credentials committed

### Performance

- Optimized database queries
- Caching where appropriate
- Lighthouse-friendly frontend
- Minimal JavaScript footprint

### Scalability & Evolution

- Easy addition of new projects or articles
- Modular architecture
- Ready for multi-language support
- Can evolve into a full personal platform
